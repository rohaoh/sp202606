# 작업 규칙 메모 (Claude 작업 시 지켜야 할 것들)

## 작업 방식
1. 새 기능/수정은 **새 날짜 폴더**로 만든다.
   - 예) `codes/2026-6-22(1)`, `(2)`, `(3)`, `(4)`, `codes/2026-6-23` …
2. 코드 수정 다 끝나면 **PR을 자동으로** 만든다. (따로 말 안 해도 디폴트)
3. 순서는 항상: **커밋 → 푸시 → 바로 PR 생성.**
4. 작업 브랜치는 `claude/fps-optimization-code-review-uo9e57` 로 고정.
5. 응답은 **한국어**로 한다.

## 기능 설계 규칙
6. 추가하는 모든 기능은 **켜고 끌 수 있게(토글)** 만든다. 앞으로도 계속.
7. **소리 관련 기능은 절대 만들지 않는다.**
8. 기록/재생은 **on-demand 방식** (사용자가 요청/arm 했을 때만 저장).
9. 물리 파일은 **직접 작성**한다. (기성 물리엔진 라이브러리 사용 금지)
   - 안정성도 유지할 것.

## 문서/기타
10. 기능 추가나 빌드 방식 변경 시 `README.md` 를 최신 정보로 갱신.
11. C++ 물리 코드 변경 시 README에 **"npm run build-addon 재실행"** 안내 포함.

## 빌드/실행 순서 (Windows)
```
cd codes/<폴더>
npm install
npm run build-addon
npm start
```
