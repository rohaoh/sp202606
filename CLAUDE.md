# 작업 규칙 (Workflow Rules)

이 레포에서 작업할 때 항상 따르는 규칙이야. (한국어로 정리)

## 1. 새 기능/수정은 새 날짜 폴더로
- 새 기능이나 수정 작업은 기존 폴더를 덮어쓰지 말고 **새 날짜 폴더**를 만들어서 진행해.
- 폴더 이름 규칙: `codes/<날짜>` 형태. 같은 날 여러 번이면 `(1)`, `(2)`, `(3)`, `(4)` 식으로 번호를 붙여.
  - 예: `codes/2026-6-22(1)`, `codes/2026-6-22(2)`, `codes/2026-6-22(3)`, `codes/2026-6-22(4)`, `codes/2026-6-23` …
- 보통은 직전 폴더 내용을 복사해서 새 폴더를 만든 뒤 거기서 수정해.

## 2. 코드 수정이 끝나면 PR 자동 생성 (디폴트)
- 따로 말하지 않아도 코드 수정이 끝나면 **기본으로 PR을 만든다.**
- 사용자가 "PR 만들지 마"라고 명시하지 않는 한 PR 생성은 디폴트 동작이야.

## 3. 커밋 → 푸시 → 바로 PR 순서로 진행
- 작업이 끝나면 **커밋 → 푸시 → 곧바로 PR 생성** 순서로 진행해.
- 푸시한 다음 바로 PR을 만든다.

## 4. 한국어로 응답
- 모든 응답은 **한국어**로 한다.
