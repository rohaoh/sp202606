## ✨ 이번 버전 핵심 (2026-6-27 Highlights)

### 1. 모델 크기를 비율 유지하며 자유롭게 조정 (신규)
- `3D Model` 섹션에 **모델 크기 (비율 유지)** 슬라이더가 추가되었습니다.
- 모델을 그룹으로 감싸 중심을 원점에 고정한 뒤 **균등 스케일**을 적용 → 크기를 바꿔도 가로·세로·높이 비율이 항상 유지되고 중심이 흔들리지 않습니다.
- **슬라이더 범위 직접 지정**: `슬라이더 최소 ×` / `슬라이더 최대 ×` 입력칸으로 슬라이더의 가동 범위를 원하는 대로 설정할 수 있습니다.
- **크기 직접 입력**: `크기 직접 입력 ×` 칸에 배율을 숫자로 입력할 수 있고, 범위를 벗어나면 슬라이더 범위가 자동으로 넓어져 따라옵니다.
- 슬라이더 / 범위 / 수치 입력은 모두 **양방향 동기화**됩니다. 기본 1.00× = 기존 정규화 크기(최대변 2.5 유닛).

### 2. STL 업로드 렌더링 수정
- 기존엔 STL 파일도 `GLTFLoader` 로 로드해 화면에 보이지 않던 문제를 수정했습니다.
- 새로 `STLLoader` 를 추가해 확장자(`.stl` / `.glb`)에 맞는 로더를 자동 선택하고, STL은 표준 머티리얼을 입힌 Mesh 로 변환해 GLB와 동일한 정렬·정규화·스케일 로직을 따릅니다.

### 3. 업로드 파일을 앱의 assets 폴더로 복사 (지연 제거)
- 내 PC 의 GLB/STL 파일을 업로드하면 메인 프로세스가 파일을 앱의 `assets/` 폴더로 복사한 뒤 로컬 HTTP 서버 정적 경로에서 로드합니다.
- blob URL 로딩 시 발생하던 **지연이 사라지고**, 복사된 파일은 다음 실행 때 프리셋처럼 **재사용**할 수 있습니다.
- 복사 불가 환경에서는 blob URL 로 자동 폴백하며, 파일명은 안전 문자로 치환해 경로 탈출을 방지합니다.

### 4. 자잘한 정리
- 파일이 없어 항상 로드 실패하던 `helicopter.glb` 프리셋을 기본 / Shape Custom / Multi-Object 드롭다운에서 모두 제거했습니다.
- `main.js` MIME 맵에 `.stl` 추가, `assets/README.txt` 갱신, `README.md` 에 2026-6-27 변경 사항 기록.

## 🧩 전체 기능

> 모든 기능은 좌측 패널/툴바의 **토글 스위치**로 개별로 켜고 끌 수 있습니다.

### 물리 시뮬레이션
- **종단속도 / 운동방정식**: `v_t = √(2mg / ρCdA)`, `ma = mg − ½ρCdAv²`
- **ISA 표준 대기**: 고도별 공기 밀도·온도·압력 자동 계산 (대류권~중간권)
- **바람**: X/Z 방향 풍속이 낙하 궤적과 착탄 위치에 반영
- **Magnus 효과**: 회전(RPM·축)에 의한 공기력 편향 계산
- **발사체 모드**: 낙하각·방위각으로 초기 속도 분해
- **지형**: 평지 / 경사면 / 수면 / 고지대 (경사면은 중력 성분 분해)
- **다중 물체**: 최대 3개 물체를 독립 시뮬레이션·그래프 비교

### 충돌 · 파괴
- **임계 기반 파괴**: 충격 압력이 재질의 유효 항복강도를 넘을 때까지 **버팀(Withstood)**, 넘어서면 점진적으로 파괴
- **타깃 두께(mm)·항복강도(MPa) 직접 입력**: 두꺼울수록 잘 버팀
- **버팀 시 바운스**: 재질이 견디면 공이 부서지지 않고 튕겨 오름(감쇠 반발)
- **충격 에너지 대시보드**: 운동에너지(KE)·운동량·충격력 표시
- **파편 / 분화구 / 먼지**: InstancedMesh 기반 고성능 파편 렌더링

### 3D 모델 (GLB / STL)
- **프리셋**: Classic_table, screwdriver, Hyperbolic (assets 폴더)
- **파일 업로드**: GLB / STL 파일을 직접 업로드 → assets 로 복사 후 정적 경로에서 로드
- **모델 크기 (비율 유지)**: 슬라이더 + 범위 직접 지정 + 수치 직접 입력으로 자유 조절 (이번 버전 신규)
- **양면 렌더링**: 노멀이 뒤집힌 모델도 한쪽 면만 잘려 보이지 않도록 보정
- **Multi-Object GLB**: 최대 3개의 추가 오브젝트도 각자 GLB 모델 사용 가능

### 시각화 · 편의
- **카메라 프리셋**: 기본 / 정면 / 측면 / 위 / 추적
- **실시간 그래프**: 재생 중 진행 지점까지 점진적으로 표시 (속도·고도·가속도·밀도 탭)
- **단위 전환**: SI ↔ km/h ↔ 제국단위(ft/s, ft)
- **궤적 라인 · 자료 표 · CSV/PNG 내보내기**
- **프리셋 저장/불러오기**(localStorage) · **설정 JSON 저장/불러오기**
- **온디맨드 녹화/재생**: REC로 arm 시에만 기록 → 히스토리에서 재생
- **설정창 폭 드래그 조절**

## 💾 설치 및 실행 (Windows)

> 처음 한 번만 Node.js, CMake, Visual Studio Build Tools(C++) 설치가 필요합니다.

```bash
cd codes/2026-6-27
npm install
npm run build-addon   # C++ 물리 엔진 컴파일 (cmake-js)
npm start
```

설치 파일(.exe)로 배포하려면:

```bash
npm run dist          # dist/ 폴더에 설치 파일 생성
```

> ⚠️ C++ 물리 코드를 수정한 경우 반드시 **`npm run build-addon` 재실행** 후 실행하세요.
> 이번 2026-6-27 버전은 C++ 물리 코드 변경은 없지만, 새 폴더에서 처음 빌드할 때는 의존성 설치 + `build-addon` 을 한 번 실행해야 합니다.
