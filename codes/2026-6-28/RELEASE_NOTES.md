### 1. 3D 모델(GLB / STL) 워크플로 전면 강화 (가장 큰 변화)
이전 버전에는 GLB/STL 모델 자체가 없었습니다. 이번 묶음에서 모델 로딩·시각화·물리 연동을 일관된 흐름으로 새로 만들었습니다.
- **GLB 모델 선택 (F17)** — `3D Model` 섹션 드롭다운에서 `assets/` 폴더의 GLB를 즉시 적용. 크기 자동 정규화, 낙하·바운스·충돌과 위치 동기화.
- **STL 업로드 지원 (신규)** — `STLLoader` 추가, 확장자(`.stl` / `.glb`)에 맞는 로더 자동 선택, STL geometry 는 표준 머티리얼 Mesh 로 변환되어 GLB와 동일한 정렬·정규화 로직을 탐.
- **프리셋 모델 3종 제공** — `Classic_table.glb`, `screwdriver.glb`, `Hyperbolic.glb` 가 `assets/` 에 동봉. (이전에는 모델 자체가 없었고 항상 로드 실패하던 `helicopter.glb` 프리셋은 이번에 정리·제거.)
- **GLB 프리셋 물리값 자동 설정** — 프리셋 선택 시 mass·area·Cd 가 모델에 맞게 자동 입력(수동 수정 가능).
- **Shape Custom (GLB)** — Shape 드롭다운에 `Custom (GLB)` 추가. 선택 시 프리셋 + 파일 업로드 UI가 바로 나타남.
- **Multi-Object GLB** — 추가 오브젝트(Object 2/3)에도 각자 GLB 모델을 지정해 같은 물리 궤도로 낙하시키기 가능.
- **모델 크기 조정 (비율 유지) — 신규** — 그룹 래핑 + 균등 스케일로 크기를 바꿔도 가로·세로·높이 비율과 중심이 그대로. `슬라이더 최소/최대 ×` 입력칸으로 슬라이더 범위를 직접 정하고, `크기 직접 입력 ×` 칸으로 배율을 숫자로 입력 가능(범위 밖이면 슬라이더 범위가 자동 확장).
- **업로드 파일을 assets 폴더로 자동 복사** — 내 PC 의 GLB/STL 업로드 시 메인 프로세스가 `assets/` 로 복사한 뒤 로컬 HTTP 서버 정적 경로에서 로드 → **blob URL 로딩 지연 제거**, 다음 실행 때 재사용 가능. (복사 불가 환경은 blob 폴백, 파일명은 안전 문자 치환으로 경로 탈출 방지.)
- **양면 렌더링(DoubleSide)** — 노멀이 뒤집힌 모델도 한쪽 면만 잘려 보이지 않도록 보정.
- **GLB 로딩 진단 토스트** — 로딩 중 / 완료 / 실패+원인+경로를 화면 토스트로 표시. 빠른 연속 선택 시 마지막 요청만 반영(시퀀스 가드).
- **패키지 빌드 GLB 안정화** — `.asar` 안에서 ESM 동적 fetch 가 실패하던 문제를 `build.asar: false` + esbuild 번들로 근본 차단. dev / 패키지 모두 동일 경로.

### 2. 물리 / 충돌 계산 확장
- **공력 가열 계산 (F16, 토글)** — 마하수 기반 정체 온도, Sutton-Graves 열유속, 복사 평형 표면 온도. 공 색상이 파랑→주황→빨강으로 변함. 오버레이/데이터 테이블에 Surface T · Heat Flux 추가.
- **파편 충돌 판정 (토글)** — `stepFragmentsJS` 를 중력·지면 / 파편 간 구 충돌 / 회전 업데이트 3단계로 재구성. 법선 방향 겹침 해소 + 탄성 충격량(반발 0.45) + 회전 교란으로 자연스러운 분산.
- **타겟 바닥에서 띄우기 (토글)** — `바닥에서 띄우기` 토글 + `띄울 높이` 슬라이더 추가. 켜면 네 개의 지지대 기둥 위에 타겟이 올라가 아래 공간이 생기고, 낙하 물체·GLB·멀티 오브젝트 전부 올라간 타겟 표면 기준으로 낙하.

### 3. 안정성 · 시각화 버그 수정
- **렌더링 전체 멈춤 수정** — GLTFLoader 가 bare specifier `'three'` 를 import 하는데 import map 이 없어 async 초기화가 중단되던 문제. import map 추가 + GLTFLoader 로드를 `try/catch` 로 감싸 실패해도 핵심 렌더링은 유지.
- **카메라 팔로업 수정** — 주 물체 착지 이후 카메라가 지면으로 급격히 줌인되던 문제. 착지 후 추적 일시 정지로 부가 오브젝트 낙하 중에도 시야 유지.
- **멀티 오브젝트 낙하 중 사라짐 수정** — 주 물체가 먼저 착지하면 나머지가 강제로 숨겨지던 문제 해결. 전체 오브젝트 최대 낙하 시간까지 재생.

### 4. 빌드 · 배포
- **arm64 지원** — Windows 플랫폼이 x64 + arm64 두 아키텍처로 확장.
- **esbuild 번들** — `renderer.js` + `three.js` + `GLTFLoader` 를 한 파일(`renderer.bundle.js`)로 번들. 런타임 동적 import 자체를 없애 패키지 앱에서 *"Failed to fetch dynamically imported module"* 오류 원천 차단.
- **`.stl` MIME 추가** — 로컬 HTTP 서버에서 STL 도 표준 MIME 으로 서빙.

## 🧩 전체 기능

> 모든 기능은 좌측 패널/툴바의 **토글 스위치**로 개별로 켜고 끌 수 있습니다.

### 물리 시뮬레이션
- **종단속도 / 운동방정식**: `v_t = √(2mg / ρCdA)`, `ma = mg − ½ρCdAv²`
- **ISA 표준 대기**: 고도별 공기 밀도·온도·압력 자동 계산 (대류권~중간권)
- **바람**: X/Z 방향 풍속이 낙하 궤적과 착탄 위치에 반영
- **Magnus 효과**: 회전(RPM·축)에 의한 공기력 편향 계산
- **발사체 모드**: 낙하각·방위각으로 초기 속도 분해
- **지형**: 평지 / 경사면 / 수면 / 고지대
- **다중 물체**: 최대 3개 물체를 독립 시뮬레이션·그래프 비교
- **공력 가열**: 정체 온도 · Sutton-Graves 열유속 · 복사 평형 표면 온도 (이번 묶음 신규)

### 충돌 · 파괴
- **임계 기반 파괴**: 충격 압력이 재질 항복강도를 넘을 때까지 **버팀(Withstood)**, 넘어서면 점진적으로 파괴
- **타깃 두께(mm)·항복강도(MPa) 직접 입력**
- **버팀 시 바운스**: 견디면 부서지지 않고 튕겨 오름(감쇠 반발)
- **충격 에너지 대시보드**: 운동에너지(KE) · 운동량 · 충격력
- **파편 / 분화구 / 먼지**: InstancedMesh 기반 고성능 렌더링
- **파편 끼리 충돌 판정** (토글, 이번 묶음 신규)
- **타겟 바닥에서 띄우기** (토글, 이번 묶음 신규)

### 3D 모델 (GLB / STL) — 이번 묶음 신규
- **프리셋**: Classic_table, screwdriver, Hyperbolic
- **파일 업로드**: GLB / STL 직접 업로드 → `assets/` 폴더로 자동 복사 후 정적 경로 로드
- **모델 크기 (비율 유지)**: 슬라이더 + 범위 직접 지정 + 수치 직접 입력
- **양면 렌더링**: 노멀 뒤집힌 모델 보정
- **Multi-Object GLB**: 추가 오브젝트도 각자 GLB 사용 가능
- **로딩 진단 토스트**: 로딩 중 / 완료 / 실패 원인·경로 표시

### 시각화 · 편의

## 💾 설치 및 실행 (Windows · x64 / arm64)

> 처음 한 번만 Node.js, CMake, Visual Studio Build Tools(C++) 설치가 필요합니다.
> arm64 환경에서는 ARM64 용 Node.js / MSVC 툴체인을 설치하세요. cmake-js 가 호스트 아키텍처에 맞춰 자동으로 빌드합니다.

```bash
cd codes/2026-6-27
npm install
npm run build-addon   # C++ 물리 엔진 컴파일 (cmake-js, x64 / arm64 모두 지원)
npm start
```

설치 파일(.exe)로 배포하려면:

```bash
npm run dist          # dist/ 폴더에 설치 파일 생성
```

> ⚠️ C++ 물리 코드를 수정한 경우 반드시 **`npm run build-addon` 재실행** 후 실행하세요.
> 새 폴더에서 처음 빌드할 때는 `npm install` + `build-addon` 을 한 번 실행해야 합니다.
